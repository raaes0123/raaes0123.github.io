Chip-8 Emulator

CHIP-8 is an interpreted programming language developed on the 1802 microprocessor. It was designed to make it easy to program small games on early systems.

Machine Description

Memory

CHIP-8 has 4096 bytes of memory. The first 512 bytes are reserved for the interpreter so programs typically start at address 0x200. The upper area (0xF00–0xFFF) is reserved for display refresh, and 0xEA0–0xEFF is used for stack and variables.

Registers

CHIP-8 has sixteen 8-bit data registers named V0–VF. VF serves as a flag register for carry, borrow, and pixel collision. The address register I is 12 bits wide. The Program Counter (PC) tracks the current instruction.

The Stack

Used for subroutine calls, originally allowing up to 12 nested calls.

Timers

  • Delay Timer counts down at 60 Hz; used for game timing.
  • Sound Timer beeps while non-zero; also counts down at 60 Hz.

Input

Input is done via a 16-key hexadecimal keypad (0-F). Three opcodes handle input:

  • Skip next instruction if key pressed
  • Skip if key not pressed
  • Wait for key press and store in register

Graphics and Sound

The display resolution is 64 x 32 pixels, monochrome. Sprites are 8 pixels wide and 1-15 pixels tall, drawn with XOR logic. VF is set on pixel collision. Sound is a simple beep controlled by the sound timer.

Opcode

CHIP-8 supports 35 opcodes, all two bytes long and stored big-endian.

Building

This project uses Raylib for graphics. Ensure Raylib is installed and update RAYLIB_PATH in the Makefile. 

RAYLIB_PATH = ./../raylib/build/raylib

Then build using:

make all

Running

After building: 

./emulator
